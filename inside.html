Question: What is the difference between map filter find?

Answer:
Map Takes a function and applies it to each element in a collection, creating a new collection of the same length with the transformed values.

Filter Takes a predicate function and applies it to each element in a collection, creating a new collection containing only the elements that satisfy the predicate.

Find: Takes a predicate function and applies it to each element in a collection, returning the first element that satisfies the predicate or undefined if none do.

Question: What is the difference between local storage and session storage and cookies ?

Answer:
Local Storage: It is a web storage method that helps us store data on the client’s computer in the form of key/value pairs in a web browser. The data is stored in local storage for a lifetime unless the user manually deletes it from the browser. It does not expire even when the user closes the window or tab. Instead, the data remains in the browser until and unless the browser's memory is cleared.

Session Storage: It is very similar to the local storage. Still, the main difference lies in the lifespan as it persists in the browser until its current tab is on. Once you close the tab or terminate it, the data on session storage also gets lost. We can also set and retrieve its data using setItem() and getItem() methods, respectively, similar to the local storage methods.

cookie: The only option that was available before HTML5 was introduced was cookies. So, storing data with it is a legacy approach to storing data on the client machine. It help us store the client-side data to enable a personalized experience for the website’s users. These are sent with requests to the server and are sent to the client on response; hence its data is exchanged with the server on every request. The servers could use the cookie data to send personalized content to users.

Question: what is event loop in javascript and how to work it

Answer: The Event Loop is a critical part of the JavaScript runtime environment, which enables JavaScript to be a single-threaded language that can handle asynchronous operations. It's responsible for handling events and executing callbacks in a non-blocking manner.
